# Reconstruction Attacks on PCEN Spectrograms

This repository contains the reproducibility code for a negative-result audit of PCEN spectrogram privacy. We evaluate oracle and blind inverse-PCEN attacks followed by Griffin-Lim and HiFi-GAN waveform reconstruction. The result is that PCEN tiles preserve enough information to reconstruct intelligible speech and speaker-identifying audio; PCEN should not be described as privacy-preserving or non-invertible.

PCEN remains useful preprocessing for acoustic event classifiers because it provides adaptive gain control and dynamic-range compression. The audit here is narrower: PCEN is not a privacy primitive, not an anonymization layer, and not a reason to store or share feature tiles as if they were safe.

## Threat Model

The attacker observes the PCEN tile consumed by the classifier: a 64-band mel representation over a 3 second, 16 kHz mono window. The attacker knows the public feature configuration, including the `librosa.pcen` defaults and the mel prescale factor.

The repository tests two recovery settings:

- `pcen_oracle`: exact inverse PCEN using the forward running-mean state. This is an upper-bound attacker.
- `pcen_blind`: iterative inverse PCEN from the PCEN tile alone. This is the realistic attacker.

Both are compared with a `mel_baseline` condition to measure whether PCEN adds privacy over plain mel features.

## Reproduction

Install the Python dependencies:

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Run the lightweight release smoke test:

```bash
python scripts/smoke_test.py
```

Check that the command-line entry points import without requiring raw datasets:

```bash
PYTHONPATH=. python tests/privacy/run_attack_eval.py --help
PYTHONPATH=. python tests/privacy/build_report.py --help
PYTHONPATH=. python tests/compute_intervals.py --help
```

Regenerate the LibriSpeech privacy audit locally:

```bash
PYTHONPATH=. python tests/privacy/data/download_librispeech.py
PYTHONPATH=. python tests/privacy/run_attack_eval.py \
    --corpus tests/privacy/data/librispeech_devclean_3s \
    --out-dir tests/privacy/reports
PYTHONPATH=. python tests/privacy/build_report.py
```

Regenerate the hidden-phrase probe audit:

```bash
PYTHONPATH=. python tests/privacy/data/synthesize_hidden_phrases.py
PYTHONPATH=. python tests/privacy/run_attack_eval.py \
    --corpus tests/privacy/data/hidden_phrases \
    --out-dir tests/privacy/reports/hidden_phrases
PYTHONPATH=. python tests/privacy/build_report.py
```

Expected outputs:

- `tests/privacy/reports/raw_results.json`
- `tests/privacy/reports/hidden_phrases/raw_results.json`
- `tests/privacy/REPORT.md`

The full attack evaluation downloads public datasets and pretrained ASR/speaker/vocoder models, so it is intentionally not part of the lightweight smoke test.

The classifier discussed in the paper is an audited case study only. It is not presented as deployment-ready, and the repository does not include raw audio, prepared tensors, checkpoints, ONNX exports, or deployment services.

## Data Availability

LibriSpeech dev-clean privacy clips are reproducible via `tests/privacy/data/download_librispeech.py`. Hidden-phrase probes are generated by `tests/privacy/data/synthesize_hidden_phrases.py` and are subject to the upstream TTS/vocoder licenses.

Raw YouTube-derived audio, violence audio, prepared tensors, and model checkpoints are not included. Data are available through public upstream datasets and local regeneration scripts where licensing permits. Generated or reconstructed privacy WAVs, if produced locally, are bounded audit artifacts and are not covered by the repository code license.

## Limitations

This is a negative-result audit, not a new privacy-preserving method. It evaluates specific PCEN settings, 3 second windows, LibriSpeech dev-clean speech, five generated hidden-phrase probes, Griffin-Lim reconstruction, and off-the-shelf HiFi-GAN reconstruction. The results should not be generalized to every audio frontend without a matching threat model and attack evaluation.

The retained classifier code supports the paper's case-study context, but the public snapshot intentionally excludes raw/private data and trained artifacts. Performance-check scripts that reference checkpoints or prepared tensors are archival helpers; they require local artifacts that are not part of this release.
